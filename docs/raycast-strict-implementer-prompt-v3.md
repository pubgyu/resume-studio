# Raycast Strict Implementer Prompt v3

`docs/raycast-strict-benchmark-v4.md`, `docs/raycast-strict-workspace-spec-v3.md`, `docs/raycast-strict-ui-tokens-v3.json`, `docs/mockups/raycast-strict-rough-v3.svg`만 기준으로 기존 기능은 유지한 채 UI를 다시 구성하라; `Raycast inspired`가 아니라 가능한 한 `Raycast strict`에 가깝게 구현하되, 공식 문서에서 확인되는 `dark utility window`, `dense row-first hierarchy`, `bottom action bar`, `compact mode`, `fast/light/frictionless editing`, `mobile home-first` 패턴만 사용하고 Raycast에 없는 warm brown 배경, peach/salmon active fill, fake command rows, support card stack, row별 상시 액션 버튼은 추가하지 말라. 로그인은 compact split utility window, 목록은 `window bar + row list + selected preview + bottom action bar`, 편집기는 `section rail + single form surface + paper preview + bottom action bar`, 모바일은 `home-first recent flow + secondary search + sticky dock`로 설계하고, active 상태는 accent fill이 아니라 neutral highlight overlay로 처리하라.
