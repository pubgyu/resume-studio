# Resume Room 서비스 개요

## 1. 서비스 정의

Resume Room은 이력서를 작성하고 저장하고, 여러 버전으로 관리하며, PDF로 내보낼 수 있는 온라인 이력서 작업 공간이다.

이 서비스는 단순한 단일 문서 편집기가 아니라 아래 흐름을 한 번에 제공하는 것을 목표로 한다.

- Google 로그인
- 사용자별 이력서 목록 관리
- 이력서별 편집 및 저장
- 템플릿 전환 및 표시 제어
- PDF 출력
- 데이터 import/export
- 모바일 편집/미리보기 전환 UX

## 2. 대상 사용자

- 개인 이력서를 여러 버전으로 관리하려는 사용자
- 저장된 이력서를 다시 열어 이어서 수정해야 하는 사용자
- 브라우저에서 문서를 작성한 뒤 PDF로 내보내려는 사용자
- 여러 사용자가 각자 자신의 이력서를 관리해야 하는 멀티 유저 서비스 환경

## 3. 핵심 사용자 흐름

### 3.1 로그인 전

- 비로그인 사용자는 `/login`으로 유도된다.
- 로그인 페이지에서는 Google 계정 로그인을 시작할 수 있다.

### 3.2 로그인 후

- 저장된 이력서가 있으면 `/resumes` 목록 페이지로 이동한다.
- 저장된 이력서가 없으면 `/resumes/new?fresh=1`로 이동해 빈 이력서부터 시작한다.

### 3.3 이력서 목록

- 저장된 이력서는 row 중심의 목록 작업 화면으로 노출된다.
- 데스크톱에서는 좌측 row 목록과 우측 선택 문서 미리보기를 함께 보여준다.
- 우측 미리보기는 HTML 문서가 아니라 PDF viewer 기준 미리보기다.
- 모바일에서는 단일 목록 shell과 하단 sticky dock 기준으로 `열기 / 복제 / 삭제 / 새 이력서` 흐름을 우선한다.
- 각 row에서 문서를 선택하고, 하단 액션에서 편집 진입, 복제, 삭제를 수행할 수 있다.
- 새 이력서를 생성할 수 있다.

### 3.4 편집기

- 데스크톱
  - 좌측: 아코디언 기반 편집 UI
  - 우측: 실시간 HTML 미리보기
  - 상단: 저장 상태, 이력서 이름, 저장/PDF/메뉴 액션
- 모바일
  - 편집과 미리보기를 한 화면에 쌓지 않고 전환형 UX로 제공
  - 하단 sticky action bar에서 `저장`과 `편집/미리보기 전환`을 우선 제공
  - PDF, import/export, 삭제 같은 보조 기능은 overflow 메뉴로 접근

편집기에서는 아래가 가능하다.

- 텍스트 입력
- 사진 업로드
- 링크 입력 및 하이퍼링크 처리
- 항목 추가/삭제
- 섹션 표시/숨김
- 세분화된 섹션 표시/숨김
- 항목 단위 표시/숨김
- 섹션 순서 변경
- 문서 템플릿 전환
- 데이터 import/export
- PDF 다운로드

## 4. 현재 제공 기능

### 4.1 인증 및 사용자 분리

- Google OAuth 로그인
- 사용자별 이력서 목록 분리
- Supabase RLS를 통한 본인 데이터만 접근

### 4.2 이력서 편집 기능

- 기본 정보, 연락처, 요약, 강점, 기술 스택
- 경력, 학력, 프로젝트, 자격증, 어학연수, 연봉, 포트폴리오
- 프로필 사진
- 날짜 기반 기간 입력
- 총 경력 자동 계산

### 4.3 문서 표시 제어

- 섹션 단위 숨김/표시
- `기본 정보`, `소개`, `핵심 강점`, `기술 스택`, `총 경력`, `경력`처럼 문서 순서 기준에 맞춘 세분화된 표시 제어
- 반복 항목 단위 숨김/표시
- 섹션 순서 드래그 정렬
- 문서 템플릿 3종
  - `default`
  - `compact`
  - `split`

### 4.4 저장 정책

- 편집 중에는 브라우저 draft(localStorage) 유지
- 저장된 이력서는 DB에 수동 저장 가능
- 기존 저장 이력서는 일정 idle 시간 이후 자동 저장
- 이탈 시 저장되지 않은 변경사항 경고

### 4.5 출력 및 데이터 이동

- PDF 다운로드
- 목록 우측 선택 문서 PDF 미리보기
- 이력서 data import/export
- JSON 기반 문서 구조 유지

### 4.6 품질 보조 기능

- 실시간 맞춤법 검사
- 다크/라이트 모드
- SEO 메타데이터 및 Open Graph
- 모바일 safe-area 대응 하단 액션 바

## 5. 제품 설계 원칙

### 5.1 편집 경험 우선

- 입력 즉시 우측 미리보기에 반영되어야 한다.
- 사용자는 저장 버튼을 누르지 않아도 현재 draft를 잃지 않아야 한다.
- 다만 최종 데이터는 사용자 계정 기준으로 명시적으로 저장되거나 자동 저장 조건을 만족해야 한다.

### 5.2 문서와 앱 셸 분리

- 앱 UI 테마는 바뀔 수 있다.
- 실제 이력서 문서 스타일과 PDF 스타일은 별도 책임으로 유지한다.

### 5.3 다중 사용자 서비스 기준

- 인증 없는 단일 브라우저 도구가 아니라 사용자별 데이터가 분리되는 서비스로 동작해야 한다.
- 데이터 접근은 항상 사용자 세션 기준으로 제한되어야 한다.

## 6. 현재 비범위 또는 후속 확장 포인트

- 협업 편집
- 공개 이력서 공유 URL
- 템플릿 추가 확대
- 서버 기반 파일 저장소 분리
- 관리자 페이지
- 결제/플랜
